Google Shopping (New Layout)

The JSON results for shopping_results are very similar between the old and the new layout. The differences are the filters parameter, pagination parameter, and the addition or removal of the categorized shopping results section.

Summary of Changes

- Sorting and filtering are now done through the shoprs parameter (tbs is no longer supported). It also works by using natural language, e.g., coffee less than $10 rather than just coffee.

- Pagination's num parameter is no longer supported.

- Pagination's start parameter is not recommended. To easily retrieve paginated results accurately, it is advisable to follow the link provided in serpapi_pagination.next.

- direct_link has to be enabled to get the link in shopping_results.

- Featured Results, Nearby Results, and Related Results are no longer available.

- Introduced new categorized_shopping_results in the API response. It is sort of a replacement of Featured Results for the new layout.

API Parameters

Advanced Filters

shoprs

Optional

The parameter defines the helper ID for setting search filters. For search filters to be properly applied, the parameter must be used in combination with the updated q parameter, which now contains the name of the filter applied next to your original query.

Example:
- Original query: Coffee
- Query after applying filter "Sugar Free": sugar free Coffee

To apply filters to your search, follow serpapi_link, which you can find in your JSON output at filters[index].options[index].serpapi_link.

To apply multiple filters, you have to follow serpapi_link for each filter individualy. Each consecutive request will contain filters that have been previously applied next to the newly added one. To remove a filter, follow its corresponding serpapi_link.

direct_link

Optional

The parameter determine if the search results should include direct link of each products. By default it is false, if you need direct link, set it to true.

This parameter only applies to the new layout (US and few other countries). Most countries have not yet switched to the new layout, so the link will be present without needing this parameter.
When using this parameter, it is expected the search will take longer than a normal search request.

Pagination

start

Optional

Parameter defines the result offset. It skips the given number of results. It's used for pagination. (e.g., 0 (default) is the first page of results, 60 is the 2nd page of results, 120 is the 3rd page of results, etc.).

For the new layout, the parameter is not recommended. To easily retrieve paginated results accurately, it is advisable to follow the link provided in serpapi_pagination.next.

num

Optional

Parameter defines the maximum number of results to return. (e.g., 60 (default) returns 60 results, 40 returns 40 results, and 100 (maximum) returns 100 results).
Any number greater than maximum number (100) will default to 100.
Any number lesser than minimum number (1) will default to 60.

For the new layout, the parameter is no longer supported.
